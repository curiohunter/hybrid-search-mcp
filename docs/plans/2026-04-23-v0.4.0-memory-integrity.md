# 2026-04-23 — v0.4.0 Memory Integrity

**Author**: autonomous execution session
**Status**: plan → executing → ship

---

## Problem statement

v0.3.0 guarantees *every* turn persists and *every* exploratory prompt gets
enriched, but treats the qa_log directory as an append-only dumpster:

- Same topic asked 3 times over 3 weeks → 3 near-duplicate qa files compete
  for the same top-10 slots.
- qa files referencing deleted / moved code remain authoritative-looking
  until age-decay pushes them down 30 days later.
- Prune = permanent `unlink()`; no safety net for a regret.

"Memory" that only appends and deletes by age is really just a FIFO queue.
True memory needs **consolidation** — dedup, staleness detection, and a
recoverable archive before deletion. This release adds those three pieces
deterministically (no new LLM cost) on top of the v0.3.0 save/retrieve
guarantees.

---

## Goals

### Must-have (ship blockers)

- **M1** qa staleness pruning — reindex deletes qa files whose *every*
  referenced source path is absent from the store DB. Mirrors the v0.3.0
  wiki orphan cleanup, applied to qa.
- **M2** semantic dedup — for each qa_log chunk, find nearest qa_log
  neighbour via the existing vector index; when cosine similarity ≥
  threshold (default 0.90) keep the newer and archive the older.
- **M4** archive tier — prunes (from auto-prune, qa-prune, M1 stale, M2
  dedup) move files into `.hybrid-search/qa-archive/YYYY/MM/` instead of
  unlinking. Archive entries are permanently removed after 30 days.
- **M5** `qa-stats` extensions — surface `active`, `archived`, `dedup'd
  last 7d`, `stale pruned last 7d`, `total ever` counters so the user
  can *see* the memory behaving healthily.
- **G_tests** every new code path has unit tests; full suite stays green
  (current 822 → expected 845+).
- **G_integrity** integration sanity: after M1+M2 run once against
  valuein, there are **zero** qa files whose refs are all gone and
  **zero** qa pairs with cosine ≥ threshold.

### Nice-to-have

- **M6** compounding bench extension — add an explicit dedup test case
  (plant same query 3×, verify M2 collapses to 1).
- **M7** `qa-restore` CLI — bring an archived qa back into qa/.
- **M_doc** CHANGELOG + README update; version bump to 0.4.0.

### Dropped from original proposal

- **M3 supersede-on-save**: originally a synchronous check during
  `qa_log.record()`. Periodic M2 covers the same ground at reindex
  time with simpler control flow. Will revisit only if post-ship
  observation shows accumulation between reindex cycles.

### Non-goals

- No LLM-based summarisation (would introduce cost + nondeterminism).
- No cross-project qa consolidation.
- No changes to the on-the-wire hook protocol or qa_log markdown format
  (additive frontmatter fields only — same backward-compat contract as
  v0.3.0).

## Architecture

### New module: `hybrid_search.memory.integrity`

All M1-M4 logic lives here so the existing `qa_log.py` / `reader.py`
boundary stays clean.

```
hybrid_search.memory.integrity
  ├── detect_stale_qa(project_root, indexed_paths) -> list[Path]
  ├── detect_semantic_duplicates(store_db, vector_index, ...) -> list[DuplicatePair]
  ├── archive_file(path, project_root) -> Path          # moves to qa-archive/
  ├── purge_old_archive(project_root, max_age_days=30)
  ├── run_integrity_pass(project_root, cfg) -> IntegrityReport
  └── stats(project_root, window_days=7) -> dict
```

`run_integrity_pass` is the single entry point the reindex pipeline calls
at the end of each run (after auto-prune, wiki cleanup). Idempotent —
running it on a clean project is a no-op.

### Archive layout

```
<project>/.hybrid-search/
  qa/                         # active, searchable
  qa-archive/                 # NEW — recoverable, not indexed, 30d TTL
    YYYY/MM/<original-stem>.md
```

Archive mirror of the active tree so restores are path-preserving.

### Integration into reindex

`cmd_reindex` current tail order:
```
1. auto_prune (age + count ceilings)   [v0.2.0]
2. wiki cleanup (orphan pages)          [v0.3.0]
   ↓ NEW
3. integrity pass (qa stale + dedup)    [v0.4.0]
4. archive purge (> 30d)                [v0.4.0]
```

Pass 3 runs BEFORE wiki cleanup would be wrong — qa stale detection
uses the same DB snapshot as wiki cleanup, so ordering is independent.
But it MUST run AFTER auto_prune so prune candidates aren't double-archived.

### Dedup algorithm (M2)

```
qa_chunks = store_db.all_chunks(node_type="qa_log")
pairs_seen = set()
victims = []

for chunk in qa_chunks:
  neighbours = vector_index.search(chunk.embedding, k=4, filter=node_type="qa_log")
  for nb in neighbours[1:]:  # skip self
    pair = frozenset((chunk.id, nb.id))
    if pair in pairs_seen:
      continue
    pairs_seen.add(pair)

    if cosine(chunk, nb) < threshold:
      continue

    older, newer = sorted_by_mtime(chunk_to_path(chunk), chunk_to_path(nb))
    victims.append(older)

archive_all(victims)
```

k=4 so we see the top-3 non-self neighbours; more than that is rare in
practice since the corpus is small. Threshold 0.90 is conservative — mem0
uses 0.90-0.95 in production. Exposed via config
`memory.integrity.dedup_threshold` (default 0.90).

### Embedding access

We do not re-embed qa files. The embeddings already live in the store DB
as part of standard indexing. The vector index (usearch) supports:
- `search(vector, k, filter_fn)` — direct ANN lookup
- retrieval of stored vectors by chunk_id for pairwise cosine

Implementation detail — if the live API surface doesn't expose cosine
directly, we compute it from `numpy.dot(a, b) / (|a|·|b|)` on vectors
fetched from the usearch index. Cheap: 2000 qa chunks × k=4 lookups =
~8k ANN ops, seconds total.

## Risk register

| Risk | Mitigation |
|---|---|
| Dedup false positives (different questions look similar) | Threshold 0.90 conservative; archive (not delete) so restorable |
| Embedding unavailable for qa chunks just written | Integrity skips chunks without embeddings; they'll be caught on next reindex |
| Archive dir accidentally indexed as source | Scanner already rejects `.hybrid-search/*` except `.hybrid-search/qa/`; qa-archive excluded by default |
| Archive eats disk | 30d retention + each file < 10 KB → 2000 × 10 KB × (30/90 churn) ≈ 6 MB ceiling |
| Breaks existing auto-prune tests | Tests covered the unlink path; new code uses archive path. Update assertions to check `qa-archive/` instead of disappearance |
| Config explosion | Single `memory.integrity` block with 2 knobs (enabled, dedup_threshold) |

---

## Rollback plan

Each T-stage is its own commit so individual reverts are possible.
`memory.integrity.enabled=false` in config disables the whole pipeline
without code changes — useful if a future regression is suspected in
production.
